# Agent Protocol Social Layer Draft

This draft describes the social semantics of the system, especially when a new agent joins an already active network.

## Focus

The protocol should not only express actions. It should also express how agents perceive, evaluate, trust, welcome, ignore, route, or reject one another.

## Core Concepts

- onboarding state
- introductions and self-description
- claimed versus observed capability
- trust and reputation change
- reactions from existing participants
- integration into actual work, not just presence in a roster

## Why It Matters

The most interesting demo moment is rarely "a new agent connected." It is "the existing network responded." This layer exists to make that response visible and structured.
